# HRMS – Employee Lifecycle & Leave Management System

## Overview
This project demonstrates an end-to-end Business Analysis implementation for a Human Resource Management System (HRMS) focused on employee lifecycle and leave management. The project covers the complete Software Development Life Cycle (SDLC), including requirement gathering, documentation, process modeling, Agile execution, and testing.

## Objectives
- Streamline employee leave management processes
- Improve visibility of leave requests and approvals
- Enable efficient tracking of employee records
- Ensure proper documentation and governance practices

## Scope
In Scope:
- Employee leave application
- Manager approval workflow
- Leave history tracking
- Employee profile management
- Notifications and reporting

Out of Scope:
- Payroll processing
- Recruitment management
- Performance evaluation modules

## Key Deliverables
- Business Requirements Document (BRD)
- Functional Requirements Document (FRD)

## Agile Implementation
The project follows Agile Scrum methodology:
- 10 Epics and 50 User Stories defined
- Acceptance Criteria for each user story
- Story Point Estimation using Fibonacci sequence
- Sprint Planning and Sprint Backlog
- Sprint Retrospective practices

## Process Design
- AS-IS Process: Represents current manual leave handling process
- TO-BE Process: Defines optimized digital workflow for leave management

## System Features
- Secure login and authentication
- Employee dashboard with leave balance
- Leave application and submission
- Manager approval and rejection workflow
- Leave history tracking
- Notification system
- Role-based access control

## Testing and Validation
- Requirement Traceability Matrix (RTM) to ensure coverage
- User Acceptance Testing (UAT) to validate business requirements
- Test cases created for key functional scenarios
